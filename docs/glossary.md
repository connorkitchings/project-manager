# Glossary

## Tracked Repo

A repository explicitly included in Project Manager's curated list.

## Status Snapshot

The normalized representation of a repo's current state after combining parsed docs and GitHub activity.

## Current Goal

The most important near-term objective a tracked repo is trying to accomplish now.

## Recent Updates

Short summary items derived from docs and recent activity that explain what changed lately.

## Attention Flag

A signal that a tracked repo may need review because it appears blocked, stale, inconsistent, or missing required documentation.

## Documentation Source of Truth

The principle that project docs describe intent and should drive the repo summary more than raw activity counts.

## Curated Include / Exclude

The v1 model where the user explicitly decides which repos appear in the dashboard.
